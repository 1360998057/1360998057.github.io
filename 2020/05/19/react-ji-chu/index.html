元素渲染

如果说React是一座大房子那元素就是React中的最不起眼也是最重要的砖头

使用

更新已经渲染的元素

React 只更新需要更新的部分

因为React DOM 会和已经渲染的DOM做比较当要更新的部分内容和渲染的DOM元素一样时不会进行更新只会更新他西药更新的部分这就是MVVM思想其核心原理是（DIFF算法)

React jsx

  • jsx是React语法的核心，javascript和html标签混写的语法叫做JSX。
  • JSX 可以很好地描述 UI 应该呈现出它应有交互的本质形式。JSX 可能会使人联想到模版语言，但它具有 JavaScript 的全部功能。
  • React 认为渲染逻辑本质上与其他 UI 逻辑内在耦合，比如，在 UI 中需要绑定处理事件、在某些时刻状态发生变化时需要通知到 UI，以及需要在 UI 中展示准备好的数据。

优点

  1. JSX执行更快，编译为JavaScript代码时进行优化。
  2. 类型更安全，编译过程如果出错就不能编译，及时发现错误。

注意点

  1. JSX必须要有根节点。
  2. 正常的普通HTML元素要小写。如果是大写，默认认为是组件。

使用要点

  1. 由HTML元素构成
  2. 中间如果需要插入变量用{}
  3. {}中间可以使用表达式
  4. {}中间表达式中可以使用JSX对象
  5. 属性和html内容一样都是用{}来插入内容

React组件

函数式组件与类组件的区别和使用，函数式比较简单，一般用于静态没有交互事件内容的组件页面。类组件，一般又称为动态组件，那么一般会有交互或者数据修改的操作。

函数式组件

函数式组件用于简单的没有交互性的组件开发

类组件

类组件是最完全的组件，函数式组件能干的类组件都可以实现，但是类组件能做的函数式组件不一定能完成.

复合组件

在组件中组件可以反复套用组件可以无限套娃

React State和Props

State

  • 当Steate改变时视图会发生变化,State 与 props 类似，但是 state 是私有的，并且完全受控于当前组件。
  • State 的更新可能是异步的。
  • 不管是父组件或是子组件都无法知道某个组件是有状态的还是无状态的，并且它们也并不关心它是函数组件还是 class 组件。

Props

  • 组件无论是使用函数声明还是通过 class 声明，都决不能修改自身的 props.
  • 父传递给子组件数据，单向流动，不能子传递给父。
